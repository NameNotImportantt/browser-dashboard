# UI Layout — browser-home-page

Описание текущей раскладки главной страницы (стартовая страница браузера).

## Общая структура

```
App (AppShell)
└── HomePage (<main>)
    ├── [декор] glow-орб (абсолютный, фон)
    ├── TopBar
    ├── Hero (приветствие + поиск + фокус)
    ├── WorkspaceTabs
    └── WidgetsGrid
        ├── TodoWidget
        ├── HabitsWidget (lazy)
        └── BookmarksWidget (lazy)
```

Корневой контейнер страницы — `<main class="container grid">`:
- ширина: `min(1240px, 92vw)`, центрирован;
- вертикальная сетка с `gap: 1rem`;
- нижний отступ: `1rem`.

Фон страницы — градиент на `body` (радиальный акцент сверху + линейный переход). Тема (`dark` / `light`) задаётся через `data-theme` на `<html>`.

## Вертикальный порядок блоков (сверху вниз)

| # | Блок | Компонент | Описание |
|---|------|-----------|----------|
| 0 | Декор | `HomePage.glowOrb` | Мягкое фиолетовое свечение, `position: absolute`, `top: -80px`, `left: 20%`, не перехватывает клики |
| 1 | Шапка | `TopBar` | Бренд, время/дата, погода, действия |
| 2 | Hero | `HomePage.hero` | Приветствие, поиск, панель «Фокус на сегодня» |
| 3 | Воркспейсы | `WorkspaceTabs` | Табы + форма создания нового воркспейса |
| 4 | Виджеты | `HomePage.widgetsGrid` | TODO, Привычки, Быстрые ссылки (3 колонки) |

## TopBar

Карточка (`card`) с трёхколоночной сеткой:

```
┌─────────────────────────────────────────────────────────────┐
│ [●] Browser Home     │  12:34  Пн, 15 июня  │  Тема  Настр. │
└─────────────────────────────────────────────────────────────┘
│ (при открытых настройках — строка на всю ширину справа)     │
│   Поисковик: [DuckDuckGo ▼]                                 │
```

| Зона | Содержимое |
|------|------------|
| Слева | Бренд: точка-градиент + текст «Browser Home» |
| По центру | `ClockDisplay` (время крупно + дата) и `WeatherChip` (температура + описание, клик — обновить) |
| Справа | Кнопка переключения темы, кнопка «Настройки» |
| Раскрывающаяся панель | Выбор поисковика (DuckDuckGo / Google), появляется под шапкой на всю ширину |

**Адаптив (< 980px):** одна колонка, элементы выстраиваются вертикально, действия и настройки выравниваются влево.

## Hero-секция

Двухколоночная сетка `1.2fr / 0.8fr` внутри карточки. Справа снизу — декоративный радиальный градиент.

```
┌──────────────────────────────────┬─────────────────────────┐
│ ЛИЧНЫЙ COMMAND CENTER            │ Фокус на сегодня        │
│ Доброе утро / ...                │ Короткий список...      │
│ Быстрый доступ к задачам...      │ • задача 1              │
│ [⌕ Поиск по интернету...] [Искать]│ • задача 2             │
│                                  │ • задача 3              │
└──────────────────────────────────┴─────────────────────────┘
```

### Левая колонка (`heroMain`)

Сверху вниз:
1. Kicker — «Личный command center» (uppercase, muted)
2. `<h1>` — приветствие по времени суток (`getGreetingByHour`)
3. Lead-текст — краткое описание страницы
4. `SearchBar` — поле поиска + кнопка «Искать» (открывает новую вкладку)

### Правая колонка (`focusPanel`, стеклянная панель `glass`)

- Заголовок «Фокус на сегодня»
- Подзаголовок
- Список до **3** незавершённых задач из активного воркспейса; если задач нет — «Нет активных задач»

**Адаптив (< 1080px):** одна колонка, фокус-панель уходит под основной блок.

## WorkspaceTabs

Карточка с двумя рядами:

```
┌─────────────────────────────────────────────────────────────┐
│ [Воркспейс 1] [Воркспейс 2] [Активный] ...                  │
│ [Новый воркспейс____________] [Добавить]                    │
└─────────────────────────────────────────────────────────────┘
```

| Ряд | Содержимое |
|-----|------------|
| 1 | Горизонтальный список табов (pill-кнопки), активный подсвечен |
| 2 | Inline-форма: input + кнопка «Добавить» |

Переключение воркспейса меняет набор задач, привычек и закладок во всех виджетах ниже.

**Адаптив (< 640px):** форма создания — одна колонка (input и кнопка друг под другом).

---

## WidgetsGrid

Три равные колонки с `gap: 1rem`. Порядок слева направо:

| Колонка | Виджет | Загрузка |
|---------|--------|----------|
| 1 | `TodoWidget` | сразу |
| 2 | `HabitsWidget` | lazy + Suspense |
| 3 | `BookmarksWidget` | lazy + Suspense |

**Адаптив:**
- `< 1080px` — 2 колонки
- `< 760px` — 1 колонка (виджеты друг под другом)

## Внутренняя структура виджетов

### TodoWidget

```
┌─ TODO ─────────────────────┐
│ [Новая задача___________]  │
│ [Приоритет ▼] [дата]       │
│ [Добавить задачу]          │
│ ─────────────────────────  │
│ ☐ Задача    [бейджи] [Уд.] │  ← drag-and-drop
│ ☑ Задача    ...            │
└────────────────────────────┘
```

- Форма: название, приоритет (low/medium/high), дата, кнопка добавления
- Список: чекбокс, название (зачёркнуто если выполнено), бейджи приоритета/даты, удаление
- Перетаскивание меняет порядок задач

### HabitsWidget

```
┌─ Привычки ─────────────────┐
│ [Новая привычка] [Добавить]│
│ ─────────────────────────  │
│ Название                   │
│ Серия: N дн.  [Отметить] [Уд.] │
└────────────────────────────┘
```

- Inline-форма добавления
- Список: название, текущая серия, кнопки «Отметить сегодня» / «Снять сегодня» и удаление

### BookmarksWidget

```
┌─ Быстрые ссылки ───────────┐
│ [Название_______________]  │
│ [https://...____________]  │
│ [Добавить ссылку]          │
│ ─────────────────────────  │
│ Ссылка (открывается в новой вкладке)  [Удалить] │
└────────────────────────────┘
```

- Stack-форма: название, URL, кнопка
- Список: кликабельная ссылка + удаление

## Состояния загрузки и ошибки (AppShell)

До готовности данных `HomePage` не рендерится:

| Состояние | UI |
|-----------|-----|
| `loading` | Центрированный текст «Загрузка...» на весь экран (`min-height: 100vh`) |
| `error` | Сообщение об ошибке + кнопка «Повторить» |
| готово | `HomePage` как описано выше |

При lazy-загрузке Habits/Bookmarks показываются fallback-карточки «Загрузка привычек...» / «Загрузка ссылок...» в соответствующей ячейке сетки.
