# Developer Guide

Internal documentation for contributing to **Browser Dashboard**.

## Prerequisites

- [Bun](https://bun.sh/)
- Node.js `20.19+` or `22.12+` (required by Vite)

## Quick Start

```bash
git clone git@github.com:NameNotImportantt/browser-dashboard.git
cd browser-dashboard
bun install
bun run dev
```

Dev server runs on Vite (default `http://localhost:5173`). Hot reload is enabled.

## Scripts

| Command | Description |
| --- | --- |
| `bun run dev` | Generate SCSS module `.d.ts` files and start Vite dev server |
| `bun run build` | Type-check with `tsc -b`, then production build |
| `bun run start` | Preview the production build locally |

## Build Output

Production build uses `vite-plugin-singlefile`. The result is **one file**:

```
dist/index.html
```

Everything ‚Äî JavaScript, CSS, inlined favicon ‚Äî is bundled into that HTML. This file is what end users download from Releases.

### Data Layer

- Database name: `browser-home-page-db`
- ORM: [Dexie](https://dexie.org/) (IndexedDB)
- Domain types live in `src/db/types/*` (one file per entity)
- Component prop types live in `src/components/<Name>/types/*`

### Styling Rules

- Only `*.module.scss` ‚Äî no global CSS files
- Colocate styles next to components: `Component.tsx` + `Component.module.scss`
- SCSS type declarations are generated by `typed-scss-modules` during `dev`

## Architecture Notes

- **State**: `useDashboardData` loads a snapshot from IndexedDB and exposes mutation handlers. Components receive props from `AppShell` ‚Üí `HomePage`.
- **Workspaces**: Most entities are scoped by `workspaceId`. Settings are global (`settings` table, key `app`).
- **Lazy loading**: `HabitsWidget` and `NotesWidget` are code-split with `React.lazy`.
- **Weather**: Open-Meteo geocoding + forecast; cache TTL is 30 minutes (`WEATHER_CACHE_TTL_MS`).
- **Search suggestions**: local history first, then Google Suggest via JSONP (`suggestqueries.google.com`).
- **i18n**: lightweight helper in `src/app/i18n.ts`; locales `ru` and `en`.

## Quality Checks

After TypeScript changes:

```bash
tsc -b
```

Style-only changes do not require a type-check run.

## Design Direction

UI follows a dark, atmospheric aesthetic inspired by the novels *The Beginning After the End* and *Shadow Slave*: magical minimalism, soft glow, purple-blue accents, clean cards, low visual noise.

## Conventions for Contributors

- Use `bun` for installs and scripts
- Do not add heavy UI frameworks without discussion
- Keep domain types out of shared `types/domain.ts` ‚Äî use entity files in `src/db/types/`
- Prefer small, focused diffs; preserve fast UI response
- Do not commit or create branches unless explicitly asked by maintainers

## Release Checklist

1. Run `bun run build`
2. Verify `dist/index.html` opens and core flows work
3. Attach `dist/index.html` to a GitHub Release
4. Update user-facing docs if behavior changed
